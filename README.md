# Channel Deck — GitHub Pages IPTV Landing Page

## এটা কী

একটা সিম্পল ল্যান্ডিং পেজ যেখানে m3u8 লিংক যোগ করলেই সেই TV চ্যানেল লাইভ প্লে হবে (HLS.js দিয়ে)। শুধু `channels.json` ফাইল এডিট করলেই হবে — কোনো কোড পাল্টাতে হবে না।

ফাইল:
- `index.html` — প্লেয়ার ও চ্যানেল লিস্ট পেজ
- `channels.json` — চ্যানেলের ডাটা (নাম, লোগো, m3u8 লিংক)

## GitHub Pages এ বসানোর ধাপ

1. GitHub এ একটা নতুন রিপোজিটরি বানান (যেমন `channel-deck`)।
2. `index.html` এবং `channels.json` — দুটো ফাইল রিপোর রুটে আপলোড করুন।
3. রিপোর **Settings → Pages** এ যান।
4. **Source** এ `Deploy from a branch` সিলেক্ট করুন, branch হিসেবে `main` (বা `master`) এবং folder `/ (root)` বেছে নিন → **Save**।
5. কিছুক্ষণ পর একটা লিংক পাবেন, যেমন:
   `https://your-username.github.io/channel-deck/`
   এটাই আপনার লাইভ ল্যান্ডিং পেজ।

## নতুন চ্যানেল যোগ করার নিয়ম

`channels.json` ফাইলটা এরকম দেখতে:

```json
[
  {
    "id": "ch01",
    "number": "01",
    "name": "Your Channel Name",
    "logo": "https://example.com/logo.png",
    "url": "https://example.com/stream/index.m3u8"
  }
]
```

- `id` — যেকোনো ইউনিক নাম (দুটো চ্যানেলের id একই হবে না)
- `number` — চ্যানেল নাম্বার (চাইলে যেকোনো কিছু লিখতে পারেন)
- `name` — চ্যানেলের নাম, এটা পেজে দেখাবে
- `logo` — চ্যানেল আইকন/লোগোর লিংক (না দিলেও চলবে)
- `url` — m3u8/HLS স্ট্রিম লিংক (এটাই সবচেয়ে গুরুত্বপূর্ণ)

নতুন চ্যানেল যোগ করতে চাইলে শুধু লিস্টে আরেকটা `{ ... }` ব্লক যোগ করুন (কমা দিয়ে আলাদা করে), commit করে push করুন। GitHub Pages কিছুক্ষণের মধ্যে আপডেট হয়ে যাবে এবং পেজ রিফ্রেশ করলেই নতুন চ্যানেল দেখাবে।

## জরুরি কথা

- শুধু `.m3u8` (HLS) ফরম্যাটের লিংক সরাসরি কাজ করবে। সাধারণ `.m3u` ফাইল (যেগুলোর ভেতরে অনেকগুলো চ্যানেলের লিংক একসাথে থাকে) ব্যবহার করতে চাইলে সেটা থেকে আলাদা আলাদা m3u8 লিংক বের করে `channels.json` এ বসাতে হবে।
- কিছু স্ট্রিম সার্ভার **CORS** ব্লক করে রাখে — সেক্ষেত্রে ব্রাউজারে স্ট্রিম প্লে নাও হতে পারে। এটা স্ট্রিম সোর্সের সেটিংসের ব্যাপার, আপনার পেজের কোডের সমস্যা না।
- বর্তমানে `channels.json` এ দুটো টেস্ট/স্যাম্পল স্ট্রিম দেওয়া আছে শুধু ডেমো হিসেবে — চাইলে এগুলো মুছে নিজের লিংক বসান।
